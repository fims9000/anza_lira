## 6) Ready Chapter-3 Block (Visualization + Math, non-overlapping with Article 1/2)

Below is a ready text block for the core chapter of Article 3.  
Focus is on *interpretable geometry visualization of AZ behavior*, not only final mask quality.

### 6.1 Core idea

In Article 3, visualization is treated as a measurable interpretation layer of the model:
1. where AZ changes segmentation errors vs baseline;
2. what direction field is selected by fuzzy regimes;
3. where anisotropy is weak/strong on the target structure.

So the section is not a repetition of “just segmentation examples” from Article 1/2.

### 6.2 Mathematical objects used in visualization

For each pixel `p` in valid region `V`, from AZ snapshot we use:

- fuzzy memberships: `mu_r(p)`, `r=1..R`;
- dominant regime:
  `r*(p) = argmax_r mu_r(p)`;
- regime confidence:
  `c(p) = max_r mu_r(p)`.

Directional angle for visualization:

- if per-pixel map exists:
  `theta(p) = theta_map[r*(p), p]`;
- otherwise from regime vector:
  `theta(p) = atan2(u_{r*,y}, u_{r*,x})`.

Signed anisotropy gain (bounded to `[-1,1]`):

`g(p) = tanh(log(sigma_u/sigma_s)) * c(p)`

where the ratio is taken for dominant regime (or dominant per-pixel regime map).  
In code this is `_safe_log_ratio(...)` + `tanh(...)` + confidence gating.

### 6.3 Error-centric difference map (Panel B)

Given baseline prediction `B`, AZ prediction `A`, GT `G`, valid mask `V`, we show sets:

- `AZ_fix_FN = (~B) & G & A & V`  (green)
- `AZ_remove_FP = B & (~G) & (~A) & V` (blue)
- `AZ_add_FP = (~B) & (~G) & A & V` (orange)
- `AZ_new_FN = B & G & (~A) & V` (red)
- `both_pred = A & B & V` (light gray)

This map is intentionally error-centric (relative to GT), so reviewer sees *what was corrected*.

### 6.4 Direction arrows along objects (Panel C)

Arrows are drawn not on full image, but on object-support skeleton:

`M_obj = (A >= tau_AZ) OR (G == 1), inside V`

then:
1. soft skeletonization (`_soft_skeletonize`, 12 iterations),
2. small-component pruning (`min_size = 24`),
3. grid-thinned anchor points.

At each point `p`, arrow orientation is model direction `theta(p)`.  
Arrow length depends on confidence:

`L(p) = 5.5 + 5.5 * c(p),   draw only if c(p) >= 0.12`

This gives dense but readable arrows *along structures*, not random background arrows.

### 6.5 Anisotropy strength map (Panel D)

Strength uses absolute signed gain:

`s_raw(p) = |g(p)|`

Robust normalization inside object mask:

`s(p) = clip((s_raw - q10)/(q95 - q10), 0, 1)`

Opacity:

`alpha(p) = 0.14 + 0.72 * s(p)`

Color ramp (colorblind-friendly): blue -> orange.  
So panel D answers: “where anisotropy is actually active on target regions”.

### 6.8 Minimal “methods paragraph” for paper body

Visualization is generated from internal AZ layer snapshots (dominant fuzzy regime, direction, anisotropy ratio) and projected to image space under valid-mask constraints.  
For each sample, we compute an error-centric difference map versus baseline, confidence-gated direction arrows on skeletonized object support, and a robustly normalized anisotropy-strength heatmap.  
This protocol links observed segmentation corrections to explicit geometric quantities instead of qualitative color overlays only.
