# Проверка: направление формируется моделью (без ручной подгонки)

Дата: `2026-04-28`

## Что проверяли

Проверка выполнена на `global_roads` (SpaceNet3 prepared test split, `39` samples) для двух запусков:

1. `results/article3_spacenet_sprint_v3_recover`  
   (`az_thesis`, фактически `fixed_cat_map` в architecture_state)
2. `results/article3_spacenet_v3_dirlearn_probe_s42_e10`  
   (`az_thesis` + `az_geometry_mode=local_hyperbolic` + `az_learn_directions=true`)

Направление берётся только из raw model snapshot (`theta_map` после выбора доминирующего fuzzy-режима), без тангенс-посткоррекции.

## Метрика коллапса направления

Использована круговая концентрация направления:

- `R1 = | mean(exp(i * theta)) |`
  - `R1 -> 1`: коллапс (почти один угол)
  - `R1 -> 0`: разнообразное поле направлений

Дополнительно:
- `mean_nonzero_bins` — среднее число непустых бинов 12-бин гистограммы углов.

## Результаты

| Run | samples | mean_R1 | mean_nonzero_bins |
|---|---:|---:|---:|
| `article3_spacenet_sprint_v3_recover` | 39 | `1.0000` | `1.00` |
| `article3_spacenet_v3_dirlearn_probe_s42_e10` | 39 | `0.8386` | `4.92` |

Интерпретация:
- в `recover` наблюдается почти полный коллапс направления в один угол;
- в `dirlearn_probe` направление стало существенно более вариативным и локально-адаптивным.

## Важное замечание по качеству сегментации

Короткий `dirlearn_probe` (`10` эпох) улучшил геометрическую вариативность направления, но по overlap-метрикам пока слабее текущего `recover` run.  
Следовательно, для финального режима нужно совместно донастроить:

1. `local_hyperbolic + learn_directions`,
2. регуляризацию геометрии,
3. длину обучения / scheduler,
4. threshold selection.

## Вывод

Требование «направление должно формироваться моделью, а не рисовальщиком» выполнено:  
последняя визуализация строится из raw `theta_map` модели.  
При этом корректная (не коллапсная) directional картина достигается именно в learnable-geometry режиме.
