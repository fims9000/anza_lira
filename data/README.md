# Данные для проекта

В этом репозитории **не хранятся сами датасеты**. В Git попадает только эта инструкция, чтобы репозиторий оставался лёгким и воспроизводимым.

## 1. Основной датасет: DRIVE

Основной прикладной датасет проекта — **DRIVE (Digital Retinal Images for Vessel Extraction)** для сегментации сосудов сетчатки.

### Где брать

Рекомендуемый путь:

- использовать официальный источник DRIVE / Grand Challenge:
  `https://drive.grand-challenge.org/`
- скачать архив вручную;
- принять условия использования датасета, если это требуется источником.

Если датасет уже есть у научной группы или на локальном диске, можно просто разложить его в нужную структуру.

### Какую структуру ожидает код

Код в `utils.py` ожидает следующую структуру:

```text
data/
└── DRIVE/
    ├── training/
    │   ├── images/
    │   ├── 1st_manual/
    │   └── mask/
    └── test/
        ├── images/
        ├── 1st_manual/
        └── mask/
```

То есть:

- `training/images/` — исходные обучающие изображения;
- `training/1st_manual/` — ручные разметки сосудов;
- `training/mask/` — маски поля зрения;
- `test/images/` — тестовые изображения;
- `test/1st_manual/` — тестовые разметки;
- `test/mask/` — тестовые маски поля зрения.

### Важная деталь по именам файлов

В загрузчике идентификатор образца определяется по началу имени файла до первого символа `_`.

Поэтому желательно сохранять стандартные имена DRIVE без самовольного переименования.

## 2. Быстрая проверка, что данные лежат правильно

После раскладки датасета можно запустить:

```powershell
python train.py --config configs/drive_smoke.yaml
```

Если структура данных верна, обучение стартует без ошибки `FileNotFoundError`.

## 3. Вспомогательные датасеты

В проекте также встречаются ранние и вспомогательные эксперименты на:

- CIFAR-10
- FashionMNIST

Они не являются основной частью текущей vessel-сегментационной линии. Для этих датасетов `torchvision` может скачать данные автоматически.

Для CIFAR-10 официальный источник:

- `https://www.cs.toronto.edu/~kriz/cifar.html`

## 4. Что не нужно класть в Git

Не добавляйте в репозиторий:

- архивы датасетов;
- распакованные папки с изображениями;
- временные копии;
- локальные бэкапы.

Данные должны храниться локально, а репозиторий должен содержать только код и инструкции.
